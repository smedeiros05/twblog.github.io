Essa é uma tradução do artigo How to Contribute to Open Source - a Guide for Technical Writers, escrito por Valentine Gatwiri.

Contribuir para um projeto opensource pode ser assustador, especialmente se você nunca fez isso antes, ou se você está acostumado a fazer documentação ao invés de código

Este guia ajudará os tech writers a começar a contribuir para projetos de código aberto e descreverá algumas das armadilhas comuns que você pode encontrar ao começar a contribuir.

Ele também destacará alguns dos benefícios de contribuir para o código aberto e fornecerá alguns recursos adicionais para começar a contribuir para o código aberto como redator técnico

O que é open source?

Código aberto é uma forma colaborativa de desenvolver e distribuir software. Pessoas de todo o mundo contribuem para projetos de código aberto adicionando recursos, corrigindo bugs, respondendo perguntas, traduzindo textos ou escrevendo tutoriais

Por quê contribuir?

Contribuir com software de código aberto é uma das melhores maneiras para os redatores técnicos e outros trabalhadores do conhecimento retribuirem e fazerem a diferença.

Você pode ajudar a melhorar o software que usa todos os dias ou pode aprender com o exemplo de outras pessoas que contribuíram antes de você. Além disso, é uma ótima maneira de conhecer pessoas com interesses comuns. E quando suas contribuições forem apreciadas, você se sentirá parte da comunidade.

Você deve estar se perguntando – quando você deve contribuir? Bem, se você é novo no código aberto, contribua no seu próprio ritmo – não se preocupe em tentar fazer muito cedo. Você encontrará aquilo com que se sente confortável.

Contribuir para o código aberto não apenas aumentará seu conjunto de habilidades, mas também permitirá que você desenvolva contatos dentro da comunidade e até mesmo crie oportunidades para trabalho freelance.

Leva algum tempo para desenvolver o conhecimento e as habilidades necessárias para se tornar um contribuidor ativo (especialmente porque a documentação geralmente requer um conjunto de habilidades diferente da programação), mas vale a pena o investimento.

Desafios para contribuir

Os redatores técnicos podem enfrentar vários desafios quando começam a contribuir para projetos de código aberto. Um dos mais comuns é não ter certeza da melhor forma de contribuir, principalmente se você ainda não sabe nada sobre o projeto.

Você também pode não ter certeza sobre onde seu trabalho deve ir ou o que se espera dos colaboradores. Existem algumas maneiras de contornar esse problema: você pode ler o arquivo README e/ou a documentação do projeto para ver se eles possuem diretrizes de contribuição. Você também pode entrar em contato com a equipe por trás do projeto e pedir ajuda.

Como contribuir para projetos Open Source

Primeiro, você deseja encontrar um projeto no GitHub com o qual deseja contribuir. Você pode ler este tutorial sobre como pesquisar no GitHub e encontrar projetos.

Em seguida, você desejará abrir o README e certificar-se de que entendeu as instruções fornecidas.

Em seguida, bifurque o repositório clicando em Fork localizado no canto superior direito da tela.

Clone o repositório bifurcado em seu computador local usando o seguinte comando:

git clone

Você verá o link do repositório ao clicar no menu suspenso do código.

Depois de cloná-lo, abra o diretório que contém seu novo fork e comece a adicionar sua contribuição.

Quando terminar, envie suas alterações no GitHub usando as seguintes instruções: 

    
        git status //displays which modifications have been staged.
        git add . // adds changes to the cloned repo
        git commit -m "changes" //performs a commit to the cloned repo
        git branch -M changes //creates a new branch
        git push -u origin changes //pushes the changes

O GitHub solicitará que você crie uma solicitação pull após enviar suas alterações para o repositório clonado. Vá em frente e crie a solicitação pull e espere que os mantenedores mesclem seu repositório clonado com o repositório principal.

Certifique-se de incorporar modificações upstream em seu repositório local se você deu fork no projeto a alguns passos atrás.

Se você encontrar um arquivo grande, instale o git-lfs se ainda não estiver instalado com o seguinte comando: brew install git-lfs

Ao baixar lentamente as versões apropriadas de arquivos enormes, a extensão Git LFS (Large File Storage), criada pela Atlassian, GitHub e alguns outros colaboradores de código aberto, diminui o impacto de arquivos grandes em seu repositório

Você também pode usar a documentação a seguir para instalar o git-lfs.

Melhores práticas para quando contribuindo com projetos open-source.

1. Identifique uma área onde você possa ajudar e encontre o projeto relevante no GitHub.

2. Leia a documentação de quaisquer projetos ou programas que possam ser semelhantes àquele em que você está interessado. Isso lhe dará uma melhor compreensão do que está envolvido na contribuição e do que outras pessoas contribuíram.

3. Pesquise os problemas rotulados como bons primeiros problemas e leia-os – normalmente são bastante fáceis de resolver.

4. Prepare seu código seguindo as diretrizes de contribuição do projeto.

5. Escreva uma solicitação pull com sua solução e explique por que ela resolve o problema em questão. Inclua links para recursos relacionados, como tutoriais, se necessário.

6. Envie sua solicitação pull para revisão. A equipe discutirá se ele deve ser aceito no repositório e atualizará você sobre a decisão.

7. Se eles decidirem não aceitar sua solicitação pull, pergunte como você poderia abordar suas preocupações para que eles reconsiderem a aceitação posteriormente.

8. Se eles aceitarem sua solicitação de pull, agradeça-lhes!

9. Continue procurando novos problemas para resolver e compartilhe seu progresso ao longo do caminho!

Quais tipos de contribuição posso fazer?

Você pode contribuir com esses projetos de várias maneiras, incluindo enviar solicitações pull para correções de bugs ou acréscimos de recursos, escrever documentação sobre como usar o software, melhorar a documentação atual, traduzir o texto para outro idioma e corrigir erros de digitação.

Você deve se envolver escolhendo um projeto no qual esteja interessado e lendo sua documentação antes de começar.

Escrever para um projeto de código aberto é uma ótima maneira de se estabelecer como um especialista em sua área e se posicionar para futuras oportunidades de freelance.

Uma coisa que os redatores técnicos devem sempre ter em mente ao escrever para projetos de código aberto é que seu público é formado principalmente por desenvolvedores. Isso significa que eles precisam de mais detalhes técnicos do que você encontraria em outros tipos de escrita.

Onde posso encontrar projetos open-source?

Há vários lugares onde você pode encontrar projetos de código aberto. GitHub é o local mais popular para projetos de código aberto, mas também existem repositórios no BitBucket, Gitlab e outros sites.

Se você tem uma ideia para um projeto de código aberto que ainda não existe, comece reunindo seu plano em um arquivo README. Se você não tiver certeza de como começar a contribuir para um projeto existente, verifique sua documentação ou leia algumas solicitações pull antes de enviar uma de sua preferência.

Exemplos de projetos

Projeto HTML5 Boilerplate é um projeto de código aberto popular para desenvolvedores web que fornece código HTML, CSS e Javascript para construir um site ou aplicativo web.

Jekyll é um gerador de sites estáticos escrito em Ruby projetado para blogs pessoais.

Documentação do React.js A documentação do React.js fornece informações oficiais sobre como usar o react.js

GitHub Pages contêm tudo o que você precisa saber sobre o GitHub.

Galaxy Project (material de treinamento) Uma coleção de materiais de treinamento sobre o projeto Galaxy, que é uma plataforma aberta baseada na web para pesquisa computacional com uso intensivo de dados que se estende além das ciências biológicas.

CNCF (Cloud Native Computing Foundation) CNCF é uma computação nativa em nuvem de código aberto que hospeda projetos como Kubernetes e Prometheus para tornar o nativo em nuvem onipresente e sustentável.

Você também pode conferir o Google Season of Docs, que fornece assistência a projetos de código aberto no aprimoramento de sua documentação e permite que escritores técnicos qualificados adquiram experiência em código aberto.
